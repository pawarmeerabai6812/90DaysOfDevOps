# Linux File System Hierarchy

## `/` - Root Directory

The root directory is the starting point of the Linux file system. Every file and directory exists under this location.

### Command

```bash
ls -l /
```

### Purpose

- View top-level directories
- Navigate using absolute paths
- Understand the Linux directory structure

---

## `/home` - User Home Directories

Contains personal directories for all users.

### Command

```bash
ls -l /home
```

### Task 1 - Check Your Home Directory

```bash
ls -la ~
```

### Purpose

- Store user files
- Keep scripts and project folders
- User-specific configuration files

---

## `/root` - Root User Home

Home directory of the root (administrator) user.

### Command

```bash
sudo ls -l /root
```

### Purpose

- Root user configuration
- Administrative scripts
- SSH keys and shell configuration

---

## `/etc` - Configuration Files

Stores system-wide configuration files.

### Command

```bash
ls -l /etc
```

### Task 2 - View Hostname

```bash
cat /etc/hostname
```

### Purpose

- System configuration
- Service configuration
- Network settings
- User and authentication settings

---

## `/var/log` - Log Files

Contains logs generated by the operating system and applications.

### Command

```bash
ls -l /var/log
```

### Task 3 - Find Largest Log Files

```bash
du -sh /var/log/* 2>/dev/null | sort -h | tail -5
```

### Purpose

- Troubleshoot application issues
- Analyze system logs
- Monitor services
- Investigate failures

---

## `/tmp` - Temporary Files

Stores temporary files created by applications.

### Command

```bash
ls -l /tmp
```

### Purpose

- Temporary storage
- Script execution
- Downloaded temporary files

---

## `/bin` - Essential Commands

Contains essential Linux commands.

### Command

```bash
ls -l /bin | head -10
```

Examples:

- ls
- cp
- mv
- rm
- cat
- bash
- echo

---

## `/usr/bin` - User Programs

Contains installed applications and utilities.

### Command

```bash
ls -l /usr/bin | head -10
```

Examples:

- python3
- git
- curl
- vim
- wget

> **Note:** On modern Ubuntu systems, `/bin` is a symbolic link to `/usr/bin`.

---

## `/opt` - Third-Party Applications

Contains manually installed software and vendor applications.

### Command

```bash
ls -l /opt
```

Examples:

- Jenkins
- Custom applications
- Vendor software

---

# Quick Reference

| Directory | Purpose | DevOps Use Case |
|-----------|---------|-----------------|
| `/` | Root directory | Absolute path navigation |
| `/home` | User home directories | Store scripts and projects |
| `/root` | Root user's home | Admin configuration |
| `/etc` | Configuration files | Configure services |
| `/var/log` | Log files | Troubleshooting |
| `/tmp` | Temporary files | Script execution |
| `/bin` | Essential commands | Core Linux utilities |
| `/usr/bin` | Installed programs | Developer tools |
| `/opt` | Third-party software | Jenkins and custom applications |

# Commands Practiced

```bash
ls -l /
ls -l /home
ls -la ~
sudo ls -l /root
ls -l /etc
cat /etc/hostname
ls -l /var/log
du -sh /var/log/* 2>/dev/null | sort -h | tail -5
ls -l /tmp
ls -l /bin | head -10
ls -l /usr/bin | head -10
ls -l /opt
```

---

# Key Takeaways

- Every file and directory in Linux starts from the root (`/`) directory.
- Configuration files are primarily stored in `/etc`.
- Logs in `/var/log` are essential for troubleshooting.
- User data is stored in `/home`, while the root user's data is stored in `/root`.
- Temporary files are stored in `/tmp` and are typically cleared after reboot.
- `/bin` and `/usr/bin` contain Linux commands and installed utilities.
- `/opt` is commonly used for third-party software installations.

---

## Conclusion

Understanding the Linux file system hierarchy is fundamental for every DevOps engineer. Knowing where configuration files, logs, binaries, and user data are stored makes troubleshooting, automation, and system administration more efficient.
